📚 Library Management System – Agile Scrum Project
📌 Project Overview

This project demonstrates the implementation of Agile methodology using the Scrum framework for managing a Library Management System. The project was carried out using Jira to create, organize, and track tasks effectively through different stages of development.

🎯 Objective

The main objective of this project is to understand how Agile practices are applied in real-world software development. It focuses on:

Creating user stories
Dividing them into smaller tasks
Managing tasks using sprints
Tracking progress efficiently
🛠 Tools and Technologies
Jira (Task Management Tool)
Agile Methodology
Scrum Framework


📋 Features (User Stories)
Book Search & Catalogue
Borrowing & Return System
Fine & Penalty Management
Member Registration


🔄 Project Workflow
Created user stories for the library system
Broke down each story into smaller manageable tasks
Assigned priorities (High, Medium, Low)
Created a sprint in Jira (LMS Sprint 1)
Moved tasks through stages:
To Do
In Progress
Done


📊 Sprint Management
All tasks were organized into a sprint (LMS Sprint 1)
Progress was tracked using the Jira board
Example progress:

✅ Book Search & Catalogue – Done
🔵 Borrowing & Return System – In Progress
⬜ Fine & Penalty Management – To Do
⬜ Member Registration – To Do
Tasks were updated regularly based on their status

✅ Results

The project successfully demonstrates how Agile Scrum helps in organizing and managing tasks in a structured way. It provides a clear understanding of:

Workflow management
Sprint planning
Task tracking using Jira
🚧 Limitations
This is a simulation project (no actual system implementation)
Limited collaboration (individual project)
